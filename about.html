About PuneRide

PuneRide is a smart bus ETA prediction system built for Pune city commuters. It uses a machine learning model trained on bus route data to predict realistic arrival times, adjusted for time-of-day traffic patterns.

Problem

PMPML commuters have no easy way to know when the next bus will arrive or how long their journey will take. This leads to missed buses, long waits, and uncertainty.

Solution

The user selects their boarding stop and destination stop from a list (or uses GPS to find the nearest stop). The system shows only buses that directly connect those two stops, predicts the wait time from the next schedule, and calculates in-bus travel time adjusted for current traffic.

How the ETA Model Works

The Linear Regression model is trained on features: route distance (km), number of stops, and traffic level (1–3). A time-of-day multiplier is applied on top — peak hours 8–10am and 5–8pm use a 1.40× delay factor; night hours use 0.85×. Dwell time per stop (1.5 min) is added to each ride estimate.

Traffic Pattern (Pune Weekday)

Traffic levels are pre-mapped by hour: midnight–5am is Low (1), 6am is Moderate (2), 7–10am is Heavy (3), 11am–3pm is Moderate (2), 4–7pm is Heavy (3), 8–10pm is Moderate (2), and 11pm–midnight is Low (1). This mirrors real Pune peak-hour patterns.

Bus Filtering Logic

When you select a boarding stop and destination stop, the system checks all buses to see which ones have both stops on their route, in the correct forward direction. Only those buses are shown. If no bus connects the two stops directly, a clear "No direct bus" message is displayed.
